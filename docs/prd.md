# 项目目标

这个项目想回答三个问题，不靠文字，靠可复现的代码和图片：

1. 银河能不能从真实星表里自然出现？（不是画一张贴图。）
2. 光污染和人眼极限星等怎么改变我们看到的天？
3. 从银河系另一个位置抬头，星空会变成什么样？

## 背景

银河不是一张贴图。它是大量真实恒星在天空中的密度分布。当我们把 Gaia 星表里的恒星按位置、亮度和颜色逐颗画出来，银河应该从恒星密度里自然出现。

这个项目把这个事实往前推一步：既然银河能从星表里自然出现，我们就可以把影响观测的变量做成可调的旋钮，看看光污染、人眼极限星等和观测位置如何一步步改变我们看到的星空。

## 目标读者

- 对天文摄影、暗空旅行、星表和可视化感兴趣的中文读者。
- 想理解"为什么暗空能看到银河、城市看不到"的普通天文爱好者。
- 想复现或二次开发 Gaia 星图渲染的开发者和 AI agent。

## 核心观点

银河一直在那里。我们看到的是被人眼、天空背景和观测视角共同过滤后的版本。

光污染抬高天空背景亮度，让低对比度的星光和银河结构淹没在背景里。人眼极限星等不是一个固定的硬件参数，是人眼和天空背景共同决定的检测边界。星座也不是宇宙里的真实结构，只是太阳附近视角形成的二维投影，走近了就散架。

## 交付内容

四件事：

第一，从真实 Gaia 星表生成全天星图，验证银河从恒星密度中自然涌现。

第二，生成暗空等级与灵敏度代价的对比视图，解释光污染如何让银河消失。

第三，生成飞行视频，展示星座散架、局部星场重投影和 Gaia 数据球边界。

第四，公开发布代码、文档和压缩结果页，让读者看得懂、开发者能复现、AI agent 能接手。

## 不做什么

- 不做测光级精确模拟。
- 不做实时交互式星图软件。
- 不伪造银河系俯视图（Gaia 可见光视差只覆盖局部数据球，无法生成真实全盘图像）。
- 不把 Gaia 大数据缓存提交到 Git。
- 不把显示的 tone mapping 伪装成纯物理模型。

## 怎样算成功

- 新读者打开 README，能在 5 分钟内理解项目在做什么。
- 开发者读完 `docs/rfc.md`，能理解数据、渲染、tone mapping 和视频管线。
- `python -m pytest tests/ -q` 在无大星表的环境中仍能通过核心测试。
- GitHub Pages 能展示压缩后的核心结果，并引导读者进入 GitHub 仓库。
- Privacy review 确认仓库不含私密数据、密钥、个人路径和不该进 Git 的大输出。
