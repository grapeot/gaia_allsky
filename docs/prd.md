# 项目目标

这个项目想回答三个问题，不靠文字描述，靠可复现的代码和图片：

1. 银河能不能从真实星表里自然出现？（不是画一张贴图。）
2. 光污染和人眼极限星等怎么改变我们看到的天？
3. 从银河系另一个位置抬头，星空会变成什么样？

第一个问题是一个验证。银河不是贴图，是大量恒星在天空中的密度分布。把 Gaia 星表里的恒星按位置、亮度和颜色逐颗画出来，银河应该从恒星密度里自然涌现。这个逻辑很直接——但跑一遍代码、亲眼看到恒星数据排列成那条横跨天空的光带，和"知道它会这样"是两回事。

这个基础反过来让后两个问题变得可操作了。既然渲染管线能真实反映恒星分布，我们就可以把平时只能靠文字描述的变量——光污染、人眼极限星等、观测位置——做成可调参数，逐步展示星空如何被重塑。

光污染抬高天空背景亮度，低对比度的星光和银河结构淹没在背景里。人眼极限星等不是固定参数，是人眼和天空背景共同决定的检测边界。很多天文科普告诉你"光污染让你看不见银河"，但很少让你看到它在哪个等级开始模糊、哪些区域先消失、这个过程是多少个等级渐变。

星座也不是宇宙里的真实结构，只是太阳附近视角形成的二维投影。走近了，它就散架了。飞行视频从太阳系出发，看着熟悉的星座瓦解、陌生的星场重组、Gaia 的数据球边界在视野里显现。

核心观点：银河一直在那里。我们看到的是被人眼、天空背景和观测视角共同过滤后的版本。

## 目标读者

- 对天文摄影、暗空旅行、星表和可视化感兴趣的中文读者。
- 想理解"为什么暗空能看到银河、城市看不到"的普通天文爱好者。
- 想复现或二次开发 Gaia 星图渲染的开发者和 AI agent。

## 交付内容

1. 从真实 Gaia 星表生成全天星图，验证银河从恒星密度中自然涌现。
2. 生成不同暗空等级下的星空对比视图，解释光污染如何让银河一步步消失。
3. 生成飞行视频，展示星座散架、局部星场重投影和 Gaia 数据球边界。
4. 公开发布代码、文档和压缩结果页，让读者看得懂、开发者能复现、AI agent 能接手。

交付物有两种成品形态，由目标分辨率决定，而非内容差异：

- **单图**：一张平面图，广州地平视角，方便浏览、分享、发公众号。正式对比图、飞行视频、以及亿级深星表的高清平面图都走这条路径。8K 以下建议都用单图。
- **HiPS 天图**：面向 8K 以上直到十亿像素级别的超大渲染。单张大图打不开也没法分享，改用 Aladin Lite 天图查看器浏览——渲染带 WCS 的 TAN 投影图，用 hipsgen 拼成 HiPS 金字塔，可 zoom in 看锐利单星、zoom out 看乳光银河。这一形态的交付物是 HiPS 金字塔目录（PNG 瓦片 + properties + MOC），可挂在静态服务器上由 Aladin Lite 加载。

## 不做什么

- 不做测光级精确模拟，定性正确即可。
- 不做实时交互式星图软件。
- 不伪造银河系俯视图（Gaia 可见光视差只覆盖局部数据球，无法生成真实全盘图像）。
- 不把 Gaia 大数据缓存提交到 Git。
- 不把显示的 tone mapping 伪装成纯物理模型——它是显示层的事，不是物理层的事。

## 怎样算成功

- 新读者打开 README，能在 5 分钟内理解项目在做什么。
- 开发者读完 `docs/rfc.md`，能理解数据、渲染、tone mapping、视频管线，以及单图和 HiPS 两种输出形态各自适用什么规模。
- `python -m pytest tests/ -q` 在无大星表的环境中仍能通过核心测试。
- GitHub Pages 能展示压缩后的核心结果，并引导读者进入 GitHub 仓库。
- 仓库不含私密数据、密钥、个人路径和不该进 Git 的大输出。
