为什么这件事有意思

普通星图告诉你天空有什么。这个实验问的是：你为什么看不到。

光污染抬高了天空背景

城市天空把背景抬亮，低对比度的星光和银河结构被埋进背景里。同一片天空还在那里，只是检测边界变了。

人眼极限不是固定数值

NELM 指裸眼极限星等，由人眼和天空背景共同决定。Bortle 1 顶级暗空能到约 7.6-8.0 等，常说的 6 等更接近普通暗天口径。

星座是视角投影

从太阳附近看，北斗像一个熟悉形状。观测者移动几十 pc（秒差距）后，近处恒星重排，星座会散架；银河带来自更大尺度的恒星分布。

前向透视：先看北斗，再转向银盘

第一段朝北斗方向移动，细线标出七颗星的位置变化。第二段从当前位置斜向飞向银心方向上方，镜头转向银盘目标点。这个视角接近普通相机的一小块视野，方向由镜头决定，适合看星座形状如何变化。预览视频无音频，看画面变化即可。

VR 全天：同一条轨迹的球面视频

equirectangular 指全天球面展开格式，可以放进手机或 VR 查看器里转头看。它和前向透视使用同一条轨迹，但保留的是整个天空；2:1 画面边缘变形来自展开方式，不是渲染错误。预览视频无音频，看画面变化即可。

边界

这个项目追求定性正确和可解释，不追求测光级精确。

必须说清楚的部分

Gaia 视差数据主要覆盖太阳附近的数据球。

飞行视频能诚实地演示星座散架、局部星场重投影和数据边界。它不能生成真正的银河俯视图，因为 Gaia 可见光视差无法覆盖整个银河盘面。人类也没有银河系俯视照片，常见俯视图来自多波段观测和模型反演。

页面里的图像还经过 tone mapping，也就是亮度映射。星等到亮度、坐标投影、视差重投影这些部分有物理锚点；把线性亮度压进 SDR 标准动态范围屏幕，则属于显示层。代码把这两层分开，避免把显示参数说成物理常数。

复现与二次开发

代码、数据获取脚本、测试和渲染命令都在 GitHub 仓库里。

正常渲染和测试不需要密钥。Gaia 缓存和完整输出留在本地，Git 里只放代码、文档、测试和网页压缩资产。

入口命令：uv venv、uv pip install -r requirements.txt、python -m pytest tests/ -q。
